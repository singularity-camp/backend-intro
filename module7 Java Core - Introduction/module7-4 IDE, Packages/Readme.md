<h1>Запуск программы</h1>

<p>Теперь скомпилируем нашу программу:</p>

<pre><code>javac HelloWorld.java</code></pre>

<p>В данной директории появился файл <code>HelloWorld.class</code>, который будет передаваться <code>JVM</code>. Последний, будет выполнять ее.</p>

<p>Для запуска, используется программа <code>Java</code>, ее аргументом будет название класса, которое нужно запустить. В нашем случае, это <code>HelloWorld</code>.</p>

<pre><code>$ java HelloWorld
Hello World!</code></pre>

<p>Поздравляю, вы вывели <code>Hello World!</code> в консоль.</p>

<h2>Пакеты</h2>

<p>Во время интенсива вы познакомились с таким компонентом в Java, как <code>class</code>. Вы создавали объекты и работали с ними. Редактор в <code>stepik</code> ограничен работой в одном файле. Реальные <code>java</code> программы состоят из сотен классов, и их необходимо делить на разные файлы, где каждый класс находится в одном файле. Это не ускорит работу программу, но улучшит читаемость вашей программы.</p>

<p>Пакет — это пространство имен, которое организует набор связанных классов и интерфейсов. Концептуально вы можете думать о пакетах как о разных папках на вашем компьютере. Вы можете хранить HTML-страницы в одной папке, изображения — в другой, а сценарии или приложения — в третьей. Поскольку программное обеспечение, написанное на языке программирования Java, может состоять из сотен или тысяч отдельных классов, имеет смысл поддерживать порядок, помещая связанные классы и интерфейсы в пакеты.</p>

<p>Платформа Java предоставляет огромную библиотеку классов (набор пакетов), подходящую для использования в ваших собственных приложениях. Эта библиотека известна как «интерфейс прикладного программирования» или сокращенно «API». Его пакеты представляют собой задачи, наиболее часто связанные с программированием общего назначения. Например, объект <code>String</code> содержит состояние и поведение строк символов; объект <code>File</code> позволяет программисту легко создавать, удалять, проверять, сравнивать или изменять файл в файловой системе; объект <code>Socket</code> позволяет создавать и использовать сетевые сокеты; различные кнопки управления объектами <code>GUI</code> и флажки и все, что связано с графическими пользовательскими интерфейсами. Есть буквально тысячи классов на выбор. Это позволяет вам, программисту, сосредоточиться на разработке вашего конкретного приложения, а не на инфраструктуре, необходимой для его работы.</p>

<p>При разработке проекта у вас могут появляться свои пакеты, которые вы сможете переносить на другой проект. К примеру:</p>

<ul>
	<li>Вы разработали пакет, который позволяет дополнять адрес в Казахстане по вхождению символов;</li>
	<li>Вы написали свой метод кодирования данных;</li>
</ul>

<p>Преимущества использования пакетов в разработке:</p>

<ul>
	<li>Переиспользование - вы можете написать реализацию в одном месте и использовать из разных мест;</li>
	<li>Организация кода - ваш код становится более читабельным;</li>
</ul>
